# Skill First

When a skill exists whose description matches the task you are about to perform, you MUST invoke it. NEVER trust that you can do it yourself. NEVER skip a matching skill because the task "seems simple" or because you "already know how."

## Enforcement

**Gate: your FIRST tool call on any new task MUST be scanning skill descriptions in your system reminders.** No Read, no Grep, no Glob, no Bash before the scan. If ANY skill's description fits — even partially — invoke it before any other action.

| Thought | Action |
|---------|--------|
| "I can just do this directly" | STOP. Scan skills first. |
| "This is straightforward, no skill needed" | STOP. Scan skills first. |
| "I know how to do this" | STOP. Scan skills first. |
| "The skill would just slow me down" | STOP. Invoke the skill. |
| "Let me explore/research first, then use the skill" | STOP. The skill handles discovery. Invoke it now. |
| "I need context before I can use this skill" | STOP. The skill IS the context-gathering step. |

A matching skill is always step zero — never step two. No pre-work before invoking it, because the skill's own workflow defines what pre-work is needed.

Skills encode decisions, constraints, and workflows that you will miss by improvising. Your confidence in the task is irrelevant — the skill exists for a reason.

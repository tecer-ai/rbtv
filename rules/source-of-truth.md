# RBTV Source of Truth

RBTV components installed into `.claude/` (skills, commands, rules, subagents) are **thin loaders or copies** generated by `install.py`. They are overwritten on every re-install. NEVER edit them directly.

## Canonical Locations

The canonical source is the RBTV repo. Its path in the target workspace is recorded in `rbtv.yaml` (`rbtv_path:` field) at the workspace root.

| What | Edit in RBTV source | Never in target |
|------|--------------------|------------------|
| Rules (`rbtv-*.md`) | `{rbtv_path}/rules/<name>.md` | `.claude/rules/rbtv-*.md` |
| Skills (`rbtv-*/SKILL.md`) | `{rbtv_path}/skills/<name>/SKILL.md` | `.claude/skills/rbtv-*/SKILL.md` |
| Commands (`rbtv-*.md`) | `{rbtv_path}/commands/<name>.md` | `.claude/commands/rbtv-*.md` |
| Subagents (`rbtv-*.md`) | `{rbtv_path}/subagents/<name>.md` | `.claude/agents/rbtv-*.md` |
| Personas | `{rbtv_path}/personas/<name>.md` | *(not installed — referenced via loaders)* |
| Workflows | `{rbtv_path}/workflows/` | *(not installed — referenced via loaders)* |
| Tasks | `{rbtv_path}/tasks/` | *(not installed — referenced via loaders)* |

## Why

Skills, commands, rules, and subagents are thin loaders or copies in `.claude/` that point back to the RBTV source at `{rbtv_path}`. Edits in `.claude/` are lost on the next re-install. The RBTV source is the only durable location.

## After Editing

For content changes (workflows, personas, tasks, SKILL.md bodies referenced by loaders), just `git pull` in the RBTV source — changes are live immediately.

For changes to what gets installed (new skill/command/rule files, module assignment), re-run the installer:

```bash
python install.py
```

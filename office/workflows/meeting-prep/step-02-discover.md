# Step 2: Discover

**Progress: Step 2 of 3** — Next: Generate

---

## STEP GOAL

Guide the user through a structured discovery conversation that surfaces everything needed to produce a high-quality cheat sheet — while adapting to what the user already provided and researching unknowns along the way.

---

## MANDATORY EXECUTION RULES

- NEVER ask questions the user already answered — acknowledge and skip
- NEVER web search without asking the user first
- NEVER treat the discovery questions as a rigid interview — follow the energy
- ALWAYS cover mandatory dimensions before moving on, even if out of order
- ALWAYS surface unknowns explicitly and ask if the user wants research

---

## MANDATORY SEQUENCE

### 1. Review Available Context

Before asking anything, review:
- What the user said in Step 1
- Any workspace files read during the context sweep
- What you already know from conversation context

Identify which discovery dimensions (from the type data file) are already covered vs. which need exploration.

### 2. Guided Discovery

Use the discovery questions from the loaded type data file as a **structured skeleton** — not a rigid questionnaire.

**Mandatory dimensions** (must be covered for every meeting type):
1. **Objectives** — What does the user want from this meeting?
2. **Relationship context** — History between the parties
3. **Strategy** — Approach, impression, what to avoid
4. **Logistics** — Date, time, format, participants

**Adaptive behavior:**
- If the user already provided information for a dimension → acknowledge it and confirm, don't re-ask
- If the user opens up on a topic → go deep, ask follow-ups, even if it means covering dimensions out of order
- If the user gives short answers → probe with more specific questions from the type data file
- Group related questions naturally — don't fire one question at a time in a list
- Present questions in batches of 2-3, not all at once

### 3. Embedded Research

During discovery, you will encounter things you don't know — companies, people, products, industry terms, competitors, market dynamics.

**When you identify an unknown:**

1. Note it explicitly: "I don't have context on {thing}."
2. Ask: "Would you like me to research {thing} before we continue?"
3. If yes: use web search, present findings concisely, and continue discovery
4. If no: ask the user to provide the context verbally

**Research triggers** — surface and offer to research when:
- The user mentions a company, person, or fund you have no information on
- The user references a product, technology, or industry term that matters for the cheat sheet
- The user's objective depends on facts you cannot verify from conversation alone
- Understanding the other party's perspective requires external information

**Research discipline:**
- Ask before every search — never search silently
- Present research findings concisely — a few bullet points, not a wall of text
- Separate facts from inference when presenting research
- If research contradicts what the user said, surface the discrepancy

### 4. Gap Check

After the conversation naturally covers the major dimensions, explicitly check for gaps:

> **Discovery summary — what I have so far:**
>
> | Dimension | Status |
> |-----------|--------|
> | Objectives | {covered / partial / missing} |
> | Relationship context | {covered / partial / missing} |
> | The other party (who they are) | {covered / partial / missing} |
> | Strategy & impression | {covered / partial / missing} |
> | Questions/doubts for the meeting | {covered / partial / missing} |
> | Things to avoid | {covered / partial / missing} |
> | Logistics | {covered / partial / missing} |
>
> {Ask about any missing or partial dimensions}

### 5. Confirm Readiness

When all mandatory dimensions are covered:

> I have enough context to generate the cheat sheet. Anything else you want to add before I write it?

HALT. Wait for the user. If they add more, incorporate it. If they're ready, proceed.

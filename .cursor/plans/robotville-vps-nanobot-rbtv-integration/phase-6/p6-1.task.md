## Execution Flow

### Phase: Understand

1. **MUST** read every file in the Context Files table above. Do not continue until all are read.
2. In the plan file, set this task's todo to `status: in_progress`.
3. Review shape.md execution log for prior task context.
4. Identify hosting requirements:
   - CLI-based deployment (Nanobot uses `exec` tool for shell commands)
   - Per-path routing: `/docs/{project}` and `/app/{project}` must be independently deployable
   - Custom domain: robotville.ai (already owned)
   - User has existing Netlify account

### Phase: Execute

1. **Evaluate Netlify vs GitHub Pages** against these criteria:

   | Criterion | Netlify | GitHub Pages |
   |-----------|---------|-------------|
   | CLI deploy via shell | `netlify deploy --dir=X --prod` | Requires git push to repo |
   | Per-path routing | Redirects/rewrites in `_redirects` or `netlify.toml` | Limited — requires repo structure |
   | Independent path deploys | Yes — deploy specific directories | No — full repo deploy only |
   | Nanobot `exec` compatibility | Direct — single shell command | Indirect — requires git operations |
   | Existing account | User has Netlify account | Would need new setup |

2. **Decide** platform based on evaluation (Netlify is strongly favored unless a blocking issue is found).
3. **Provision site** on selected platform — create site, connect robotville.ai domain.
4. **Configure DNS** — point robotville.ai to hosting provider (Netlify DNS or CNAME).
5. **Verify** domain resolves and HTTPS is active.

**Discovery Handling:**
- If simple discovery (<5 min to resolve): Resolve immediately, document in shape.md
- If complex discovery: Add new task to plan, notify user of task addition

### Phase: Validate

1. Verify `https://robotville.ai` resolves (even if to a placeholder page).
2. Verify HTTPS certificate is active.
3. Verify CLI deploy tool is identified and documented.

### Phase: Close

1. Append execution entry to shape.md (never modify existing entries).
2. **MUST** present a brief summary to the user (max 2000 characters) of what was done. Do not mark complete until the user approves.
3. After user approval, set this task's todo to `status: completed` in the plan file.
4. Notify user of completion and any plan changes.

# Task p6-2: Configure Deploy Credentials on VPS

## Goal

Configure the Nanobot VPS service account with credentials to deploy content to the hosting provider (Netlify or GitHub Pages) via CLI, so Nanobot can publish project documents on user command.

## Execution Flow

### Phase: Understand

1. **MUST** read every file in the Context Files table above. Do not continue until all are read.
2. In the plan file, set this task's todo to `status: in_progress`.
3. Review shape.md — specifically the p6-1 entry for which hosting platform was selected.
4. Identify the CLI tool and authentication method for the selected platform.

### Phase: Execute

1. **If Netlify:** Install `netlify-cli` on VPS, configure auth token for the `nanobot` service account. Store token in `/etc/robotville/nanobot-gateway.env` following existing secret hygiene patterns. Link CLI to the site created in p6-1.
2. **If GitHub Pages:** Configure SSH deploy key with write access to the website repo. Add to `nanobot` service account's SSH config following existing deploy-key patterns.
3. **Verify** the `nanobot` service user can execute the deploy command successfully (test with a placeholder file).
4. **UPDATE** `server-env-template.md` with the new credential entry and configuration guidance.

2. Verify credential is stored securely (restricted permissions, not in committed files).
3. Verify a test deploy reaches the live site.

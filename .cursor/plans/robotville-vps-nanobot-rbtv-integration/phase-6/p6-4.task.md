## Execution Flow

### Phase: Understand

1. **MUST** read every file in the Context Files table above. Do not continue until all are read.
2. In the plan file, set this task's todo to `status: in_progress`.
3. Review shape.md for hosting platform, deploy method, and site structure decisions.
4. Understand the two deployment modes:
   - **Raw download mode:** Project output files (markdown) are made available as direct file downloads at `robotville.ai/docs/{project-name}/`
   - **HTML-structured mode:** Project output files are converted to a structured HTML page at `robotville.ai/docs/{project-name}/` with navigation, formatting, and table of contents

### Phase: Execute

1. **Design** the deploy command interface:
   - Command trigger (e.g., `deploy docs` or `publish docs`)
   - User selects project and mode (raw or HTML)
   - Nanobot reads project output files from `_bmad-output/{project-name}/`
   - Nanobot executes deploy via CLI (`exec` tool)
2. **CREATE** deploy workflow/script that Nanobot invokes:
   - Collects project output files from the project's output folder
   - For raw mode: copies markdown files to `/docs/{project-name}/` with an index
   - For HTML mode: converts markdown to HTML with navigation and deploys to `/docs/{project-name}/`
   - Deploys via the hosting CLI configured in p6-2
   - Updates the `/docs/` index page to include the new project
3. **CRITICAL RULE:** Deploy executes ONLY when user explicitly commands it. Never automatic. Never triggered by framework completion.
4. **Store** deploy workflow files in `_mobile/ops/scripts/` or appropriate location.

4. Verify `/docs/` index page updates to include the deployed project.

### C. `runtime/js/text-format.js` â€” `getActiveEditElement()` (context â€” do NOT change)

```js
function getActiveEditElement() {
  const active = document.activeElement;
  if (!active) return null;
  if (active.getAttribute("contenteditable") !== "true") return null;
```

### D. New helper `scaleWholeBox(box, dir)` (add to `text-format.js`)
Proportional whole-box font scaling (spec Â§5.3 / D10):
- `base = parseFloat(getComputedStyle(box).fontSize)`; `newBase = Math.max(8, Math.round(base + dir*2))`; `factor = newBase / base`.
- Capture `beforeHtml = box.innerHTML` AND `beforeSize = box.style.fontSize`.
- Set `box.style.fontSize = newBase + "px"`. For EVERY descendant of `box` that carries an explicit inline `font-size` (`descendant.style.fontSize`), multiply it: `parseFloat(d.style.fontSize) * factor` (rounded, min 8) â†’ `d.style.fontSize`. Non-explicitly-sized text inherits the new base, so all ratios are preserved.
- Capture `afterHtml = box.innerHTML` AND `afterSize = box.style.fontSize`.
- **Undo correctness (binding):** `commands.format` captures ONLY `innerHTML`; the box's OWN `font-size` change would not be undone by it. Therefore `push` a command that restores BOTH â€” build it inline in this module (you already import `push`):

  ```js
    undo() { const b = byId(selInfo.hypId); if (b) { b.style.fontSize = beforeSize; b.innerHTML = beforeHtml; } },
    label: "format",
  });
  ```
  (`scaleWholeBox` is called from `apply`, so `selInfo.hypId`/`box` are in scope; pass what it needs.)

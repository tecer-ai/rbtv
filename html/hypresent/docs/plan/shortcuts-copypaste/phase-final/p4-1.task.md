# Task p4-1: Independent cold-verifier done-gate (C1–C9) + compat regression

## Goal

Buy "done" with independent evidence: an independent **cold verifier** — given ONLY the contract criteria and the running app, never the builders' tests/claims/sheets — re-exercises C1–C9 at the fidelity floor and returns its OWN evidence sheet; and the compatibility regression set is confirmed green. This is the orchestrated-done-gate requirement (`rbtv-done-gate` § Orchestrated Dispatches): builder-graded sheets are insufficient.

## Execution Flow

### Phase: Understand
1. Confirm every build task (`p1-1`, `p2-1`, `p2-2`, `p3-1…p3-4`) is committed on `master` and every per-phase checkpoint was approved.

### Phase: Execute
1. **Dispatch one independent cold verifier** (Agent-tool Claude, opus-tier, fidelity-floor capable). Give it ONLY: the C1–C9 criteria (verbatim from the three specs / design §11), how to launch the real app (`python server/server.py 127.0.0.1 8799`), the real deck `tecer-gsmm-introduction.html`, and the fixtures `tests/e2e/fixtures/flow-grow.html` / `grid-healthy.html`. Do NOT give it the builders' tests, claims, or the per-phase sheet. It re-exercises each criterion in a headed browser with real input and returns its OWN sheet (criterion | gesture | observed | evidence file | verdict), captures written during ITS exercise.
2. Run the **compat regression set**, reading each exit code off the UN-piped process: `python -m pytest tests/e2e/test_r3_delete.py tests/e2e/test_r8_font_size_repeat.py tests/e2e/test_r7_alignment.py tests/unit -q` → all pass (esp. `test_r3_delete::test_no_keyboard_delete`).

### Phase: Validate
1. Compare the cold verifier's sheet against the per-phase builder rows. **Any builder/verifier mismatch = this dispatch FAILS return-verification** → back to Build (re-open the relevant phase), then re-run p4-1.
2. Consolidate into the final evidence sheet: every C1–C9 row `held` (or surface each `failed`/`unexercisable`/`held-surprising` explicitly with its concrete reason).

### Phase: Gate
1. Present the consolidated sheet path + a one-line verdict summary + every non-`held` row surfaced explicitly.
2. **MUST** append the Human Review Presentation block. **HALT** — this evidence feeds the final checkpoint; the owner decides acceptance of any `held-surprising`/`unexercisable` row.

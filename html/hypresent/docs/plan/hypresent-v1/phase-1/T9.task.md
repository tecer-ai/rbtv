# Task T9: Serializer — Standalone Chrome-Free HTML Emit

## Goal

CREATE the serializer: clone the document, strip ALL `hyp-`/editor chrome and `data-hyp-*` and injected inline styles, restore original `contenteditable`, re-embed the comment JSON island, run the node-count guard, and emit a standalone HTML string. NO whole-document DOMPurify/sanitizer pass — the document's own scripts/handlers/IIFE are preserved by NOT touching them (the opened file is the user's own, not untrusted; DOMPurify cannot distinguish injected from native scripts by provenance).

## Execution Flow

### Phase: Understand
1. MUST read every Context File.
2. Mark T9 in progress in BOTH locations.
3. Confirm the STRICT order: clone → strip ALL `hyp-` artifacts → restore contenteditable → re-embed island → node-count guard → emit. There is NO DOMPurify/sanitizer step. The document's own `<script>` (REPORT IIFE), inline handlers, `<style>`, SVG, and native `data-*` survive because the strip touches ONLY `hyp-` namespaced things.

### Phase: Execute
1. CREATE `runtime/js/serializer.js` with `serialize()→htmlString` implementing `01` §5 exactly. Do NOT import DOMPurify.
2. Strip pass (the entire chrome-removal mechanism): remove every `hyp-`prefixed id/class node (handles, overlays, selection ring, markers, the injected runtime `<script type="module">`, any injected `<style>`); remove every `data-hyp-*` attribute (incl. `data-hyp-id`); remove `hyp-` class TOKENS from elements that also carry native tokens (drop the `class` attr only if emptied); remove injected inline styles the editor added; restore each element's original `contenteditable` from the registry. Per-element COLOR overrides (D6) are document content — KEEP them.
3. Re-embed: write the comment store JSON into `<script type="application/json" id="hyp-comments">` at end of `<body>` (the one intentional `hyp-` survivor), AFTER the strip so it is not removed.
4. Guard: compute the pre/post node-count delta; the expected delta = removed `hyp-` chrome nodes + the re-embedded island. A delta outside that band → abort with an `error` event (never emit a damaged file).
5. Emit `<!doctype html>\n` + cloned `documentElement.outerHTML`.

**Discovery Handling:** simple → resolve + shape.md; complex → add task, notify.

### Phase: Validate (T9-scoped — empty comment store, features not yet built)
1. Chrome-free: serialized DECK and REPORT have ZERO `hyp-`/`data-hyp-` artifacts (no `#hyp-comments` yet — store empty).
2. Doc JS retained: serialized REPORT keeps its own `<script>` IIFE byte-for-byte; DECK gains no `<script>`.
3. Valid standalone HTML: the output loads in a fresh page without layout collapse.
4. (Visual/reflow fidelity, comment-island round-trip, and re-editability are T18's gates — NOT verified here.)

### Phase: Close
1. Append decision/discovery to shape.md (standalone) or defer (orchestration).
2. `human_review: required` → emit the Human Review Presentation block (format + flag criteria per `3-resources/tools/rbtv/workflows/planning/templates/plan-task-microstep-template.md` § Human Review Presentation and `.../data/plan-creation-rules.md` § Human Review Flag Criteria) — serializer is the editor-leakage and document-JS-preservation gate.
3. Mark T9 complete in BOTH locations after acceptance passes.

# Task T8: Unified Undo/Redo History Stack

## Goal

CREATE the unified history stack: a single linear command/inverse stack covering ALL operation types (text, format, resize, move, color, comments), with cursor, redo-tail truncation, and a `history-changed` event.

## Execution Flow

### Phase: Understand
1. MUST read every Context File.
2. Mark T8 in progress in BOTH locations.
3. Confirm: a `command` is `{do(), undo(), label}`; `push` runs `do()`; feature modules NEVER mutate the document outside a command.

### Phase: Execute
1. CREATE `runtime/js/history.js`: `push(cmd)` (executes `do()`, truncates any redo tail), `undo()` (runs `undo()` of the current entry, moves cursor back), `redo()` (re-runs `do()`, moves cursor forward), `state()→{cursor,canUndo,canRedo}`; emit `history-changed` after every mutation.
2. Ensure the stack is operation-type-agnostic — it stores opaque command objects only.

**Discovery Handling:** simple → resolve + shape.md; complex → add task, notify.

### Phase: Validate
1. Push synthetic commands of mixed types; undo N restores the captured pre-state for each; redo N re-applies.
2. After an undo, a new push truncates the redo tail; `state()` reports `canRedo:false`.

### Phase: Close
1. Append decision/discovery to shape.md (standalone) or defer (orchestration).
2. Mark T8 complete in BOTH locations after acceptance passes.

# Review 01 — RBTV Slide-Library Convention v1 (Adversarial Contract Review)

**Reviewer role:** adversarial reviewer (break-it-before-consumers-do).
**Date:** 2026-06-06.
**Artifacts:** `slide-library/docs/convention-spec.md` (576 lines + ADX-1/2/3), `slide-library/docs/fixture-spec.md` (767 lines).
**Live sources verified:** `5-workbench/tecer-biz/slide-library/` — `assemble.py`, `manifest.md`, `presets.md`, `base.html`, `CLAUDE.md` (read-only, zero writes). Recon docs read for context only; live source outranked them.
**Method:** every load-bearing claim traced to live source `file:line`; the minimal-YAML round-trip and the base64 image payloads executed in a sandbox; the manifest/preset parsers replicated verbatim from `assemble.py` and run against the spec's own examples.

---

## VERDICT: **NO-GO**

Three BLOCKERs make a non-reasoning executor produce a wrong engine or a false-green fixture, and one makes DT5 structurally unachievable as written. None is deep; all are mechanical and fixable in a focused revision pass. Counts: **5 BLOCKER · 7 MAJOR · 6 MINOR.**

The single most dangerous defect is **RV-1 (round-trip break)**: the spec mandates the engine WRITE `deviations:` as a YAML block list in `as-built.md`, but the "tecer-proven minimal YAML reader" the spec leans on (and that the fixture seed entry assumes) **cannot parse that list back** — it silently mangles every multi-line emitted entry. What the engine writes ≠ what the engine can read. That breaks DT5 reproduction and corrupts the migrated tecer log.

# Task p2-2: Cheat-sheet overlay + shell shortcut listener

## Goal

Add a **"?"** topbar button + a cheat-sheet overlay listing every shortcut, and a shell-side keyboard listener that handles the same combos when focus is in the shell chrome (and receives `"shortcut"` events the runtime forwards). ## Acceptance criteria (owner-observable)
- A **"?"** button sits in the topbar (next to Redo); clicking it OR pressing `Ctrl+/` opens an overlay listing all shortcuts grouped **Text / Components / Editing**; `Esc` or an outside-click closes it (C5).
- With focus in the shell chrome, `Ctrl+B`/`Ctrl+I` format the selected box, `Ctrl+Alt+C` opens the comment composer, `Ctrl+Del` deletes the selected component.
- When the runtime forwards `emit("shortcut",{action})`, the shell opens the comment composer (`"comment"`) or the overlay (`"show-shortcuts"`) — this is how C3 works when focus is in the iframe.

## New module — `app/js/shell/shortcuts-help.js` (CREATE)
Export `createShortcutsHelp() → { open() }`. On first `open()`, build a modal overlay once and append to `document.body`; subsequent calls toggle it visible. Structure: a full-screen scrim (class e.g. `shortcuts-scrim`) + a centered card (`shortcuts-card`) titled "Keyboard shortcuts", listing three groups with `<kbd>` keys:
- **Text:** Bold `Ctrl+B`, Italic `Ctrl+I`, Bigger `A+`, Smaller `A−`
- **Components:** Copy `Ctrl+C`, Paste (float) `Ctrl+V`, Paste into layout `Ctrl+Shift+V`, Delete `Ctrl+Del`
- **Editing:** Comment `Ctrl+Alt+C`, Undo `Ctrl+Z`, Redo `Ctrl+Shift+Z`, Show shortcuts `Ctrl+/`
Close on: click on the scrim (outside the card), and `Esc` (attach the keydown on open, remove on close — do not leak listeners). Pure shell UI; import nothing from `runtime/`.

## `app/js/main.js` edits
### A. Import (top of file, with the other shell imports)
```js
import { createShortcutsHelp } from "/app/js/shell/shortcuts-help.js";
```
### B. Module-scope handle (so `ensureBridge` and `DOMContentLoaded` share it). Add near the other module-level `let` declarations (e.g. after `let isEditingNow = false;`):
```js
let shortcutsHelp = null;
```
### C. In `ensureBridge(iframe)`, add a `"shortcut"` subscription alongside the other `bridge.on(...)` calls:
```js
  bridge.on("shortcut", (payload) => {
    const action = payload && payload.action;
    if (action === "comment") document.getElementById("comment-btn")?.click();
    else if (action === "show-shortcuts") { if (shortcutsHelp) shortcutsHelp.open(); }
  });
```
### D. Inside the `DOMContentLoaded` handler (after the existing button wiring), instantiate the help, wire the "?" button, and add the shell keydown listener:
```js
  shortcutsHelp = createShortcutsHelp();
  document.getElementById("shortcuts-btn")?.addEventListener("click", () => shortcutsHelp.open());

  document.addEventListener("keydown", (e) => {
    if (!(e.ctrlKey || e.metaKey)) return;
    const t = e.target;
    const inField = t && (t.tagName === "INPUT" || t.tagName === "TEXTAREA");
    const k = e.key.toLowerCase();
    if (k === "/") { e.preventDefault(); shortcutsHelp.open(); return; }
    if (inField) return;                       // don't hijack typing in shell inputs
    if (e.altKey && k === "c") { e.preventDefault(); document.getElementById("comment-btn")?.click(); return; }
    if (e.key === "Delete") { e.preventDefault(); document.getElementById("delete-btn")?.click(); return; }
    if (!e.altKey && k === "b") { e.preventDefault(); if (bridge) bridge.command("format", { op: "bold" }).catch(() => {}); return; }
    if (!e.altKey && k === "i") { e.preventDefault(); if (bridge) bridge.command("format", { op: "italic" }).catch(() => {}); return; }
  });
```
(`#comment-btn` / `#delete-btn` already carry the full guarded flows; clicking them reuses those. `bridge` is the module-level handle.)

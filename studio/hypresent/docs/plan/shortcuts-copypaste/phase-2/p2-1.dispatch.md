## Design тАФ `shortcuts.js` uses dependency injection (NO import from runtime-main)

`runtime/js/runtime-main.js` imports `shortcuts.js` (to wire it); `shortcuts.js` MUST NOT import from `runtime-main.js` (that would be an ES-module cycle). So `shortcuts.js` exports an init function that runtime-main calls with the action closures:

```

```

`initShortcuts` attaches ONE `document` `keydown` listener (inside the iframe) that:
- `Ctrl/Meta+B` (no Alt) тЖТ `preventDefault()`; `handlers.bold()`.
- `Ctrl/Meta+I` (no Alt) тЖТ `preventDefault()`; `handlers.italic()`.
- `Ctrl/Meta+Alt+C` тЖТ `preventDefault()`; `handlers.requestComment()`.
- `Ctrl/Meta+/` тЖТ `preventDefault()`; `handlers.requestShortcutsHelp()`.
- `Ctrl/Meta+Delete` тЖТ ONLY when a component is selected AND NOT editing: `preventDefault()`; `handlers.deleteComponent()`. Otherwise let the browser handle it (native word-delete while editing).
- Editing test: `const ed = document.activeElement; const editing = !!(ed && ed.getAttribute && ed.getAttribute("contenteditable") === "true");`
- NEVER bind plain `Delete`/`Backspace`.

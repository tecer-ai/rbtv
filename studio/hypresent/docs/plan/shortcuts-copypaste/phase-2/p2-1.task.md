# Task p2-1: In-iframe keyboard shortcut router

## Goal

Add the in-document keyboard shortcuts inside the iframe: `Ctrl+B`/`Ctrl+I` (formatting), `Ctrl+Del` (delete component, same guards as the toolbar), and forward shell-owned actions (`Ctrl+Alt+C` comment, `Ctrl+/` show-shortcuts) to the shell via `emit("shortcut", {action})`. Behaviour source of truth: `../specs/shortcuts-spec.md` (bindings table, capture architecture, the bridge-event contract). Copy/paste keys are added later (p3-4) — do NOT add them here.

Serialization note: `runtime/js/runtime-main.js` is shared — its order is `p1-1 → p2-1 → p3-4`. p1-1 has already landed when you run; build on the current file.

## Acceptance criteria (owner-observable)
- `Ctrl+B`/`Ctrl+I` with the iframe focused apply formatting (whole-box if selected, highlighted text if editing — the p1-1 path).
- `Ctrl+Del` with a component selected (not editing) deletes it; blocked while editing and on the last region; **plain `Delete`/`Backspace` stay inert**.
- `Ctrl+Alt+C` and `Ctrl+/` with the iframe focused reach the shell as `emit("shortcut", {action:"comment"|"show-shortcuts"})`.

## Design — `shortcuts.js` uses dependency injection (NO import from runtime-main)

`runtime/js/runtime-main.js` imports `shortcuts.js` (to wire it); `shortcuts.js` MUST NOT import from `runtime-main.js` (that would be an ES-module cycle). So `shortcuts.js` exports an init function that runtime-main calls with the action closures:

```
export function initShortcuts(handlers) // handlers: { bold, italic, deleteComponent, requestComment, requestShortcutsHelp }  (copy/pasteFloat/pasteLayout added in p3-4)
```

`initShortcuts` attaches ONE `document` `keydown` listener (inside the iframe) that:
- `Ctrl/Meta+B` (no Alt) → `preventDefault()`; `handlers.bold()`.
- `Ctrl/Meta+I` (no Alt) → `preventDefault()`; `handlers.italic()`.
- `Ctrl/Meta+Alt+C` → `preventDefault()`; `handlers.requestComment()`.
- `Ctrl/Meta+/` → `preventDefault()`; `handlers.requestShortcutsHelp()`.
- `Ctrl/Meta+Delete` → ONLY when a component is selected AND NOT editing: `preventDefault()`; `handlers.deleteComponent()`. Otherwise let the browser handle it (native word-delete while editing).
- Editing test: `const ed = document.activeElement; const editing = !!(ed && ed.getAttribute && ed.getAttribute("contenteditable") === "true");`
- NEVER bind plain `Delete`/`Backspace`.

### B. Refactor the delete handler so the keyboard path reuses its guards. Current block (locate by exact text):
```js
  register("delete-element", (payload) => {
    if (!payload || !payload.hypId) {
      throw new Error("delete-element: missing hypId");
    }
    // Edit-active guard (V3-S10): never delete while a text edit is active.
    const active = document.activeElement;
    if (active && active.getAttribute && active.getAttribute("contenteditable") === "true") {
      return { blocked: "editing" };
    }
    const el = byId(payload.hypId);
    if (!el) {
      return { blocked: "not-found" };
    }
    // Last-region guard (V3-S7): never delete the only remaining top-level region.
    if (el.parentElement === document.body) {
      const bodyRegions = Array.from(document.body.children).filter((c) =>
        c.getAttribute("data-hyp-id")
      );
      if (bodyRegions.length <= 1 && bodyRegions[0] === el) {
        return { blocked: "last-region" };
      }
    }
    // V3-S8: reanchor after BOTH do() and undo() so deleted-element threads go unanchored
    // on delete AND re-anchor on undo (the Undo path runs cmd.undo() only).
    const cmd = makeDeleteCommand(payload.hypId);
    historyPush({
      do() { cmd.do(); reanchorComments(); },
      undo() { cmd.undo(); reanchorComments(); },
      label: cmd.label,
    });
    clear();              // selection cleared → the observer unmounts the Moveable (V3-S9)
    return { deleted: payload.hypId };
  });
```
Extract its body into a local function `function deleteComponentById(hypId) { … return {blocked|deleted} }` (same logic verbatim, parameter `hypId` in place of `payload.hypId`) defined inside `boot()`, and make the handler delegate: `register("delete-element", (payload) => { if (!payload || !payload.hypId) throw new Error("delete-element: missing hypId"); return deleteComponentById(payload.hypId); });`. Behaviour MUST be byte-identical for the shell path.

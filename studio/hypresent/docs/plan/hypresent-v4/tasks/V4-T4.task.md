## Goal (R12 — Alt held during resize anchors at the element CENTER on ANY element)
Moveable's `fixedDirection` is the resize-anchor primitive. Default = the edge OPPOSITE the dragged handle (one-sided). Setting `fixedDirection:[0,0]` pins the CENTER → symmetric. Moveable snapshots `fixedDirection` at resizeStart, so the assignment is start-only. Two edits in `interaction.js`. `applyVisualResize` is UNCHANGED (S3-6 — the R10 dist-write is anchor-agnostic; under the center anchor `e.dist` arrives pre-doubled, so `beforeRect+dist` still lands the dragged edge under the cursor while the box grows 2Δ).

`moveable` is the module-level `let` in `interaction.js`, in scope in both handlers. `e.inputEvent` is the native pointer event Moveable forwards (the same object `onDrag` already reads for `clientX`).

LOCATE the `onResizeStart(e)` handler. After its existing capture block (where `beforeRect`/`beforeSizing` are captured), INSERT:

if (moveable) { moveable.fixedDirection = (e.inputEvent && e.inputEvent.altKey) ? [0, 0] : false; }

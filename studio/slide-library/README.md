# RBTV Slide-Library

The RBTV slide-library convention is a small, deterministic contract for self-describing slide libraries. A library is a single folder that contains everything a machine or a human needs to assemble a presentation deck: a manifest of fragments, named presets, a vendored assembly engine, a design system, and an as-built log. The convention separates reusable library inputs from per-deck client data—slides travel as generic, token-bearing fragments, and every client-specific fill happens in the assembled output. This makes the same library consumable by a graphical builder, a cold autonomous agent, or a direct engine invocation with identical results.

The canonical artifacts live in this repo. `docs/convention-spec.md` is the normative contract—library layout, manifest schema, asset resolution, language rules, validation table, the archive convention, and the cold-agent interface. The canonical engine is `engine/assemble.py`; the canonical archive tool is `engine/archive.py` (archive/restore superseded fragments into an `archive/` folder the engine never reads—see spec § 1.3); `engine/install-engine.py` re-vendors both into a target library and syncs the `engine_version` stamp. `tests/fixture-library/` is the synthetic conformance library that exercises every normative feature and is the first gate for any engine change.

A library adopts the convention by creating the required files (`library.json`, `manifest.md`, `presets.md`, `as-built.md`, `base.html`, `theme.css`, `slides/`, `assets/`), copying the engine into its root, and writing `CLAUDE.md` from the template in the spec (with an optional `AGENTS.md` mirror for Codex agents). See [`docs/convention-spec.md`](docs/convention-spec.md) for the complete, authoritative details.

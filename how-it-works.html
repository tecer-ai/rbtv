SKILLrbtv-commit — Guided Git Commit ▼

Description: Commit changes with diff analysis, concern-based grouping, conflict detection, and conventional message generation.

Triggers: User says "commit", "save to git", "commit it" or task ends and changes must be persisted.
Logical Step-by-Step:
  1. Analyze Changes: Run git status and git diff to identify staged, unstaged, and untracked
  2. Group by Concern: Files related to same feature/fix form a cluster. One commit per cluster, never unrelated bundles
  3. Confirm Plan: Present clusters, files, and proposed messages. Wait for user confirmation
  4. Fetch and Check Remote: Run git fetch and check for remote commits
  5. Conditional - No remote changes:
    • For each cluster: git add {files} (NEVER git add -A)
    • Gate: Verify that staged list = plan exactly
    • Commit with confirmed message
  6. Conditional - With remote changes:
    • If dirty: git stash
    • git pull
    • git stash pop
    • If pop fails (conflict): Present conflicts, wait for user resolution
    • If pop succeeds: Run tests if they exist, then commit each cluster
  7. Push: Only if user requested
Critical Conditionals:
  • If strange staged files exist before commit: Unstage or disclose, NEVER commit silently
  • If stash pop fails: STOP, don't commit. Present conflicts, resolve with user
  • If upstream exists: Fetch first, pull if remote has commits
Workflow: {rbtv_path}/core/workflows/commit/workflow.md
Loader: core/skills/commit/SKILL.md
SKILLrbtv-web-searching — Rigorous Web Search ▼

Description: Three interaction modes: Preview (metadata), Extract (clean content via Defuddle), Research (multi-source with source scoring).

Triggers: User provides URL, requests topic research, or task requires web information.
Mode Selection:
Signal Mode
Need only title/description (identification, reading) Preview
User provides URL to read/analyze full content Extract
User requests topic research, comparison, data collection Research
Preview Mode (Fallback Chain):
  1. Try: defuddle parse <url> -p title and -p description
  2. If fails: oEmbed API (Twitter, YouTube, noembed generic)
  3. If fails: WebFetch with meta-scoped prompt
  4. If fails: Ask user manually
Extract Mode:
  1. Use Defuddle CLI: defuddle parse <url> --md
  2. If Defuddle empty/error: Fallback to WebFetch
Research Mode (Rigorous):
  1. Identify Topic: Use structured input or most recent user question
  2. Load Standards: Read {rbtv_path}/core/workflows/web-search/data/web-research-standards.md
  3. Search: WebSearch for multiple sources, defuddle to extract content
  4. Evaluate Sources: Score = (Authority + Trustability + Topic Match) / 3. Discard TS < 6
  5. Group by Domain: Multiple links from same domain = ONE source
  6. Apply Data Standards: Every claim with inline citation, explicit units, Fact vs Analysis vs Speculation
  7. Format Output: Legend mandatory, Citation format, Sources Discarded section
Critical Rule: NEVER present information without verified source. TS < 6 = FORBIDDEN. Every research response MUST include legend, scored citations, and Sources Discarded section. 
Workflow: {rbtv_path}/core/workflows/web-search/workflow.md
Loader: core/skills/web-searching/SKILL.md

Orchestration

Purpose: Long-horizon work — structured planning, execution via tiered sub-agents, long source mining. Multi-agent work repeatable with per-phase review.
SKILLrbtv-orchestrating — General Orchestration Engine ▼

Description: Single skill routing any multi-agent work body. Thin loader → always-loaded core-protocol → JIT cards (intake, routing, dispatch-wrapper, verification, halt-recovery, state).

9 Invariants (Always True):
  1. Never execute work yourself: Conductor routes, packages, gates, reconciles — NEVER writes the DELIVERABLE
  2. No dispatch without artifact + state spine: Task file + 3-file spine on disk are mandatory
  3. Disk = truth: Verify EVERY return against git status/log. Message is hint, disk is truth
  4. Halts are the loop: doubt_policy: halt is normal path, not failure. Never blind-retry
  5. Evidence must be plausible: Mechanical tripwires: commit hash in git log, WALL_MS plausible
  6. Ask budget at every intake: EVERY run: ask if swap model to save spend
  7. Follow ask-the-user taxonomy: Interrupt ONLY at genuine executive decisions + defined halts
  8. Adaptive brain floor: Weaker conductor → escalate every judgment point
  9. Depth cap ≤ 2: Max 2 conductor levels (top + 1 CLI sub-conductor)
Situation Table (Router):
Situation Card to Open
Run opening (goal or plan) intake.md
Task ready for dispatch routing.md
Worker chosen dispatch-wrapper.md
Return arrived (any status) verification.md
Quality boundary reached verification.md
Worker didn't finish cleanly (halt, BLOCKED, crash) halt-recovery.md
State file must be written state.md
Run resuming state.md (rebuild from state-capsule.md)
Dispatch to CLI model (kimi, codex, claude-cli, qwen) models/{model}/manual.md
Installed Model Packages (Availability Line): 
Model packages installed: claude-cli, codex, kimi
Absent: qwen

RULEcaveman — Compressed Chat Mode ▼

Description: Forces all chat prose into caveman-speak. Same technical substance, radically fewer words. Intensity scales with response length.

Intensity Spectrum:
Level Style Example
Normal Full sentences, connectives, polish "Your component re-renders because you're passing an inline object as a prop."
Caveman (lite) Fragments, no articles, abbreviations "Inline obj prop → new ref → re-render. useMemo."
Caveman (dense) Pure fragments, arrows, max compression "New ref each render → shallow compare fail. useMemo fix."
Auto-Clarity Escape: Security warnings and irreversible action confirmations ALWAYS revert to full, clear sentences. Caveman resumes immediately after.
Turning Off: Say "stop caveman" or "normal mode".

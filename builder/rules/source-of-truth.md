# RBTV Source of Truth

RBTV components installed into `.claude/` (skills, commands, rules, subagents) are **thin loaders or copies** generated by `install.py`. They are overwritten on every re-install. NEVER edit them directly.

## Canonical Locations

The canonical source is the RBTV repo. Its path in the target workspace is recorded in `rbtv.json` (`rbtv_path:` field) at the workspace root.

The repo is module-first — `<module>` is the component's owning module folder (declared in `{rbtv_path}/admin/install/module-manifest.json`).

| What | Edit in RBTV source | Never in target |
|------|--------------------|------------------|
| Rules (`rbtv-*.md`) | `{rbtv_path}/<module>/rules/<name>.md` | `.claude/rules/rbtv-*.md` |
| Skills (`rbtv-*/SKILL.md`) | `{rbtv_path}/<module>/skills/<name>/SKILL.md` | `.claude/skills/rbtv-*/SKILL.md` |
| Commands (`rbtv-*.md`) | `{rbtv_path}/<module>/commands/<name>.md` | `.claude/commands/rbtv-*.md` |
| Subagents (`rbtv-*.md`) | `{rbtv_path}/<module>/subagents/<name>.md` | `.claude/agents/rbtv-*.md` |
| Personas | `{rbtv_path}/<module>/personas/<name>.md` | *(not installed — referenced via loaders)* |
| Workflows | `{rbtv_path}/<module>/workflows/` | *(not installed — referenced via loaders)* |
| Tasks | `{rbtv_path}/<module>/tasks/` | *(not installed — referenced via loaders)* |

## Why

Skills, commands, rules, and subagents are thin loaders or copies in `.claude/` that point back to the RBTV source at `{rbtv_path}`. Edits in `.claude/` are lost on the next re-install. The RBTV source is the only durable location.

## After Editing

For content changes (workflows, personas, tasks, SKILL.md bodies referenced by loaders), just `git pull` in the RBTV source — changes are live immediately.

When a skill, command, rule, or subagent is created, deleted, or renamed, update `{rbtv_path}/admin/install/module-manifest.json` to reflect the change, then re-run the installer:

```bash
python install.py
```

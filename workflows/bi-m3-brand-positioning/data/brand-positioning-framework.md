# Brand Positioning Framework

**Purpose:** Define the specific place a brand occupies in the customer's mind relative to alternatives using a positioning statement and perceptual map.

---

## The Positioning Statement Template

A positioning statement is not a tagline — it is an internal strategic document that aligns every downstream decision.

**Template:**
> "For **[target segment]** who **[need/job]**, **[brand]** is the **[category]** that **[key benefit]**, unlike **[primary alternative]** which **[key limitation]**."

### Element Definitions

| Element | What It Is | What It's NOT |
|---------|------------|---------------|
| **Target Segment** | Specific person/org in specific situation | Broad demographic label |
| **Need/Job** | Functional or emotional job in customer language | Product feature need |
| **Category** | Recognizable category customers already budget for | Made-up jargon category |
| **Key Benefit** | Single most important outcome (purpose made tangible) | Feature list |
| **Primary Alternative** | What target most likely uses today | Weakest competitor |
| **Key Limitation** | Real acknowledged shortcoming of alternative | Straw man weakness |

---

## The Perceptual Map

A two-dimensional visualization that plots competitors along strategically relevant axes.

### Common Dimension Pairs

| Axis 1 | Axis 2 | Best For |
|--------|--------|----------|
| Simple | Powerful | Productivity/SaaS tools |
| Self-serve | Full-service | Service businesses |
| Affordable | Premium | Consumer products |
| Generalist | Specialist | Professional services |
| Automated | Manual | Process tools |
| Individual | Team | Collaboration software |

### Map Construction

1. Choose 2 dimensions meaningful to target (they evaluate alternatives on these)
2. Plot 5-8 competitors including "do nothing" option
3. Plot your intended position
4. Identify white space
5. Verify white space has real customer need (empty is not automatically valuable)

---

## Four Validation Tests

Every positioning statement must pass all four:

### 1. Uniqueness Test
> "Could any competitor credibly make the same claim?"

If yes → revise benefit, alternative, or category frame.

### 2. Credibility Test
> "What evidence supports the benefit claim?"

List: capabilities, testimonials, architecture, expertise. If aspirational, document gap-closing plan.

### 3. Relevance Test
> "Does the benefit directly address the JTBD primary job?"

Compare to customer's functional/emotional progress. If not recognized → benefit is irrelevant.

### 4. Consistency Test
> "Does it align with Golden Circle, Brand Prism, and Lean Canvas?"

Document and resolve any contradictions.

## Positioning Inputs Brief Structure

Gather from upstream frameworks:

| Section | Content | Source Framework |
|---------|---------|------------------|
| **Target** | Who we position for (segment, role, situation) | Lean Canvas, Working Backwards |
| **Need** | What job/problem we address | JTBD, Lean Canvas Problem |
| **Benefit** | What we promise (purpose made tangible) | Golden Circle Why + How |
| **Category** | What kind of thing we are | Golden Circle What |
| **Alternatives** | Who we position against | M2 competitive landscape |
| **Differentiators** | Why we are different | How principles, Unfair Advantage, Brand Prism |

---

## Prerequisites

Before starting Brand Positioning:

- Golden Circle complete (Why provides benefit, How provides differentiation, What provides category)
- Brand Prism complete (physique, personality, relationship shape competitive frame)
- M1 Lean Canvas complete (UVP, Segments, Unfair Advantage)
- M2 TAM/SAM/SOM complete (competitive landscape provides alternatives)

---

## Pitfalls

### 1. Feature-Based Positioning
"The only platform with real-time sync and 50+ integrations" — fragile because competitors copy features.

**Instead:** Write benefit as customer outcome, not product capability.

### 2. Unrecognizable Category
"Predictive collaboration intelligence" — forces market education before selling.

**Instead:** Start with category target already searches for, add modifier.

### 3. Straw Man Alternative
Choosing weak alternative with obvious limitation — feels dishonest.

**Instead:** Choose strongest relevant alternative with real, hard-to-fix limitation.

### 4. Skipping Perceptual Map
Writing statement without visualizing competitive space risks invisible overlap.

**Instead:** Build map before finalizing statement.

### 5. Publishing Positioning As Messaging
Positioning statement is internal strategy, not customer-facing copy.

**Instead:** Use Messaging Architecture to translate positioning into audience-appropriate language.

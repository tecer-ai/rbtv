# Lean Canvas Framework Knowledge

**Purpose:** Reference knowledge for the Lean Canvas 9-block business model hypothesis framework.

---

## Framework Overview

Lean Canvas encodes business model hypotheses across nine blocks. Each block is a hypothesis to be validated, not a fact.

### The 9 Blocks

| Block | Question | Key Output |
|-------|----------|------------|
| 1. Problem | What are the top 3 problems? | Problem statements (customer language) |
| 2. Customer Segments | Who experiences these problems? | Primary segment + early adopters |
| 3. Unique Value Proposition | Why are you different and worth attention? | One-sentence promise |
| 4. Solution | What are the top 3 features? | Feature-to-problem mapping |
| 5. Channels | How will you reach customers? | Acquisition + delivery channels |
| 6. Revenue Streams | How will you make money? | Pricing model hypothesis |
| 7. Cost Structure | What are main costs? | Fixed vs variable costs |
| 8. Key Metrics | How will you measure success? | 3-5 stage-appropriate metrics |
| 9. Unfair Advantage | What can't be easily copied? | Defensibility thesis |

---

## Block Guidance

### Problem Block

- Express in **customer language**, not feature gaps
- Include consequences of not solving
- Reference situations from JTBD and PSF
- Consider "do nothing" as implicit alternative

### Customer Segments Block

- Define primary segment: role, org type/size, geography
- **Early Adopters**: narrower slice who feel pain most acutely
- Mark secondary segments as "future" only
- Must be specific enough to name 3 real examples

### Unique Value Proposition Block

Formula: "For **[segment]** who struggle with **[problems]**, **[product]** is a **[category]** that **[benefit]**, unlike **[alternatives]** which **[shortcoming]**."

- Focus on **change in customer's world**, not product features
- Derive from PR/FAQ headline and JTBD job statement
- Must differentiate from at least 2 named alternatives

### Solution Block

- Top 3-5 features that deliver on UVP
- Each maps to specific Problem or job
- Express as **what changes for customer**
- Consider integration with existing workflows

### Channels Block

- **Discovery channels**: How they first hear about you
- **Activation channels**: How they experience value
- Match to JTBD behaviours and existing digital touchpoints
- Consider self-serve vs sales-assisted based on segment

### Revenue Streams Block

- Choose pricing model suited to segment's buying process
- Consider: per-seat, usage-based, freemium, transaction fees
- Align with value created (pay for outcome vs access)
- Note budget authority and procurement path

### Cost Structure Block

- **Fixed costs**: salaries, core infrastructure, compliance
- **Variable costs**: hosting per user, API calls, support load
- Identify **step functions** (e.g., "hire CS at 20 customers")
- Note which costs matter for unit economics

### Key Metrics Block

- 3-5 metrics tied to job and UVP
- Classify as **leading** (early signals) or **lagging** (outcomes)
- Must be measurable at current stage
- Avoid vanity metrics (raw signups without context)

### Unfair Advantage Block

- Must be **hard to copy or buy**
- Valid sources: proprietary data, network effects, distribution partnerships, deep expertise
- **Reject**: "great team", "first mover", generic advantages
- Inform from Competitive Landscape analysis

## Assumption Tagging Convention

Each assumption should be tagged for M2 validation:

| Tag | Block | Example |
|-----|-------|---------|
| P1, P2, P3 | Problem | "Problem frequency >2x per week" |
| CS1, CS2 | Customer Segments | "Market has >1000 companies fitting profile" |
| UVP1 | UVP | "Customers perceive us as differentiated" |
| SOL1 | Solution | "Feature X solves problem Y" |
| CH1, CH2 | Channels | "SEO can deliver 1000 leads/month" |
| REV1 | Revenue | "Customers will pay $X/seat" |
| COST1 | Costs | "Infrastructure costs <20% of revenue" |
| MET1 | Metrics | "Activation rate predicts retention" |
| UA1 | Unfair Advantage | "Data moat grows with usage" |

## Validation Checklist

**Canvas Completeness:**
- [ ] All 9 blocks populated
- [ ] Canvas Brief defines scope, segment, and stage
- [ ] Version tag present (e.g., "Lean Canvas v0.1 – Idea Stage")

**Traceability:**
- [ ] Each Problem traces to JTBD or PSF
- [ ] Each Solution traces to Problem
- [ ] UVP consistent with PR/FAQ headline
- [ ] Unfair Advantage informed by Competitive Landscape

**Assumptions:**
- [ ] At least 3 assumptions tagged for validation
- [ ] No assumption marked as "validated" without evidence

### Phase: Execute

CREATE `_config/.cursor/rules/bmad-rbtv-skill-standards.mdc` with:

**Frontmatter:**
- `description`: "Structure and naming standards for Claude Skills (SKILL.md files)"
- `globs`: target `_config/.cursor/skills/**` and `workflows/build-rbtv-component/templates/ide-loader-template.md`
- `alwaysApply`: false

**Body content (lean, mandatory language per atomic files rule):**
1. SKILL.md naming — case-sensitive, exact spelling, no variations
2. Folder naming — kebab-case, no spaces, no capitals
3. `name` field MUST match parent folder name exactly
4. `description` MUST include what the skill does AND when to use it (trigger conditions)
5. No `README.md` inside skill folders
6. No XML angle brackets (`< >`) in YAML frontmatter
7. Size: RBTV thin loaders under 35 lines; general skills under 5,000 words
8. Optional fields (compatibility, metadata, allowed-tools, license) — when to use each
9. Optional subdirectories (scripts/, references/, assets/) — allowed but not required for thin loaders
10. Progressive disclosure — keep SKILL.md lean, link to references/ for heavy docs

### Phase: Validate

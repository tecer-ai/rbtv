# Task p2-2: Add Prior Context Section to Framework Templates

## Execution Flow

### Phase: Understand

1. **MUST** read every file in the Context Files table above. Do not continue until all are read.
2. In the plan file, set this task's todo to `status: in_progress`.
3. Review shape.md execution log — especially p2-1's ownership mapping.
4. Use Task tool with `subagent_type='explore'` to locate all framework template files (typically in `workflows/bi-m{N}-{framework}/templates/`).
5. Review the PRD's proposed Prior Context format.

### Phase: Execute

1. For each framework template that is NOT the first framework in its milestone (first framework owns concepts, has no prior context to reference):
   - INSERT `## Prior Context` section near the top of the template (after frontmatter/header, before framework-specific content)
   - Section structure:
     ```
     ## Prior Context
     **Builds on:** [List of prior frameworks whose output this framework extends]
     **Inherits:** [Concepts from prior frameworks used as-is — do not restate]
     **Adds:** [Net-new concepts this framework introduces]
     ```
   - Populate based on the content ownership mapping from p2-1

### Phase: Validate

1. Verify every non-first framework template has a Prior Context section.
2. Verify first frameworks in each milestone do NOT have Prior Context (they are concept originators).
3. Verify "Inherits" lists match the ownership mapping — each inherited concept points to the correct owning framework.

### Phase: Close

1. Append execution entry to shape.md (never modify existing entries).
2. **MUST** present a brief summary to the user (max 2000 characters) of what was done. Do not mark complete until the user approves.
3. After user approval, set this task's todo to `status: completed` in the plan file.

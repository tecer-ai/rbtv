## Execution Flow

### Phase: Understand

1. **MUST** read every file in the Context Files table above. Do not continue until all are read.

3. Review shape.md execution log for prior task context.
4. Review the merge audit to understand all differences and proposed merge strategies.

### Phase: Execute

For each step file (step-01 through step-09):

1. Start from the investor version as the base (or whichever has more complete content per the audit).
2. Apply merge strategies from the audit:
   - Replace hardcoded investor/client references with `{pitch_type}` conditionals
   - Use conditional blocks for content that differs by type:
     ```markdown
     **If pitch_type = investor:**
     {investor-specific content}

     **If pitch_type = client:**
     {client-specific content}
     ```
   - Keep shared content as-is
3. Update all internal step references to point to `pitch-creation/steps-c/` paths
4. Preserve all knowledge file references (`html-patterns.md`, `html-components.md`, `pitch-reference.md`)
5. CREATE each merged step file at `workflows/pitch-creation/steps-c/step-{NN}-{name}.md`

**Step file naming must match existing convention:**
- step-01-init.md
- step-02-context-gather.md
- step-03-narrative.md
- step-04-data-layer.md
- step-05-research-prompt.md

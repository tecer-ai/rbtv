# Task p1-1: CREATE tools-manifest.csv

## Goal

Create `_bmad/rbtv/tools-manifest.csv` with columns `id`, `skill_path`, `subagent_path`, `description` by scanning installed skills and subagents.

---

## Context Files

**MUST read every file in the table below before any execution phase.** Do not proceed to Phase: Understand until all are loaded and read.

| File | Purpose |
|------|---------|
| `.cursor/plans/rbtv-tools-manifest/shape.md` | Scope, key decisions, manifest column structure |
| `_bmad/rbtv/rbtv-manifest.csv` | Existing flat manifest — cross-reference for completeness |

---

## Execution Flow

### Phase: Understand

1. **MUST** read every file in the Context Files table above. Do not continue until all are read.
2. In the plan file (the `.plan.md` in this plan's directory), set this task's todo to `status: in_progress`.
3. Review shape.md execution log for prior task context.
4. List all skill directories under `_bmad/rbtv/.cursor/skills/bmad-rbtv/` — each subfolder name is a tool id.
5. List all subagent files under `_bmad/rbtv/.cursor/agents/bmad-rbtv/` — each `.md` filename (without extension) is a tool id.
6. Read `_bmad/rbtv/rbtv-manifest.csv` to cross-reference tool coverage.

### Phase: Execute

1. For each discovered tool id, determine:
   - `skill_path`: relative path to `SKILL.md` (e.g., `_bmad/rbtv/.cursor/skills/bmad-rbtv/{id}/SKILL.md`), or empty if no skill exists.
   - `subagent_path`: relative path to agent `.md` file (e.g., `_bmad/rbtv/.cursor/agents/bmad-rbtv/{id}.md`), or empty if no subagent exists.
   - `description`: Extract from skill's SKILL.md description or agent file frontmatter. Keep to one sentence.
2. Write `_bmad/rbtv/tools-manifest.csv` with header row `id,skill_path,subagent_path,description` and one row per tool, sorted alphabetically by id.
3. Verify row count matches total unique tool ids discovered.

**Discovery Handling:**
- If simple discovery (<5 min to resolve): Resolve immediately, document in shape.md
- If complex discovery: Add new task to plan, notify user of task addition

### Phase: Validate

1. Verify deliverable meets goal statement.
2. Confirm every tool from `rbtv-manifest.csv` appears in the new `tools-manifest.csv`.
3. Confirm no duplicate ids exist.
4. Confirm all `skill_path` and `subagent_path` values point to files that exist.

### Phase: Close

1. Append execution entry to shape.md (never modify existing entries).
2. **MUST** present a brief summary to the user (max 2000 characters) of what was done. Do not mark complete until the user approves.
3. After user approval, set this task's todo to `status: completed` in the plan file.
4. Notify user of completion and any plan changes.

## Execution Flow

### Phase: Understand

1. **MUST** read every file in the Context Files table above.
2. Set this task's todo to `status: in_progress` in the plan file.
3. Review shape.md execution log for prior task context.
4. Map existing functions to replacements:
   - `_convert_mdc_frontmatter_to_claude` → DELETE (no longer needed)
   - `_convert_agent_frontmatter_to_claude` → DELETE (no longer needed)
   - NEW: `_convert_claude_rule_to_mdc` — converts `.md` rule (with `paths:` YAML array) → `.mdc` rule (with `globs:` comma string, add `alwaysApply:`)
   - NEW: `_convert_claude_agent_to_cursor` — converts Claude agent frontmatter (`permissionMode: plan` → `readonly: true`, strip unsupported fields like `tools`, `hooks`, `memory`, `maxTurns`, `mcpServers`, `skills`)

### Phase: Execute

1. DELETE function `_convert_mdc_frontmatter_to_claude` from installer.
2. DELETE function `_convert_agent_frontmatter_to_claude` from installer.
3. CREATE function `_convert_claude_rule_to_mdc(content: str) -> str`:
   - Parse YAML frontmatter from Claude `.md` rule.
   - Convert `paths:` (YAML array) → `globs:` (comma-separated string).
   - Add `alwaysApply: true` (default for RBTV rules).
   - Return content with `.mdc`-compatible frontmatter.
4. CREATE function `_convert_claude_agent_to_cursor(content: str) -> str`:
   - Parse YAML frontmatter from Claude agent.
   - Convert `permissionMode: plan` → `readonly: true`.
   - Strip Claude-only fields: `tools`, `hooks`, `memory`, `maxTurns`, `mcpServers`, `skills`, `bypassPermissions`.
   - Return content with Cursor-compatible frontmatter.

### Phase: Validate

1. Verify old functions are removed.
2. Verify new functions exist and have correct signatures.
3. Verify conversion logic handles edge cases: missing `paths` field, no frontmatter, empty frontmatter.

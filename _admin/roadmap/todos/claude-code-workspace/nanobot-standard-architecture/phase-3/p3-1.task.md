## Execution Flow

### Phase: Understand

1. **MUST** read every file in the Context Files table above.
2. Set this task's todo to `status: in_progress` in the plan file.
3. Analyze both installer scripts to identify:
   - Shared logic (BMAD config updates, output path writes, help catalog)
   - IDE-only logic (.cursor/.claude config, MCP merge, .vscode, .cursorignore)
   - Admin-only logic (path substitution, reinforcement append, .gitignore, config prompts)
4. Design shared function architecture.

### Phase: Execute

1. Restructure `_config/install-rbtv.py` with mode selection (CLI argument or auto-detect).
2. Extract shared functions: BMAD config updates, output-path normalization, help catalog entry.
3. Implement IDE mode preserving all current `install-rbtv.py` behavior.
4. Implement admin mode preserving all current `install-admin-rbtv.py` behavior.
5. Add stub for sync mode (implemented in p3-5).
6. Update `CLAUDE.md` installer documentation to reflect unified script.

**Discovery Handling:**
- If shared logic is more complex than anticipated: document in shape.md, proceed with best design.
- If admin installer has undocumented behavior: document in shape.md.

### Phase: Validate

1. Verify IDE mode produces identical output to original `install-rbtv.py`.
2. Verify admin mode produces identical output to original `install-admin-rbtv.py`.
3. Verify shared functions are callable from both modes.
4. Verify sync mode stub exists but is not yet functional.

### Phase: Close

1. Append execution entry to shape.md.
2. Present summary to user. Do not mark complete until user approves.
3. After approval, set todo to `status: completed`.

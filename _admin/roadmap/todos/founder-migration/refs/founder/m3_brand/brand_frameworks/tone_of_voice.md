# Tone of Voice Guidelines

**Purpose:** Task-based guide for defining how the brand speaks across all contexts, using tone dimensions, do/don't examples, and context-specific adjustments in M3 Brand Step 7.

**Context:**
Read `brand_process.md` before starting. Execute in M3 Brand Step 7: Define Verbal Identity. This is the last brand framework in M3. MUST complete Messaging Architecture (Step 6) first. Uses Brand Archetypes (character), Brand Prism (personality facet), and Messaging Architecture (message examples) as primary inputs.

---

## Framework Overview

Tone of Voice defines how a brand sounds — not what it says (that is Messaging Architecture's job), but how it says it. The methodology uses tone dimensions represented as slider scales (for example, formal-to-casual, serious-to-playful) to create a precise, reproducible verbal identity. This approach originates from editorial style guide practice, popularized in the digital product world by teams like Mailchimp, whose Content Style Guide became the reference implementation for tech companies. The core principle: a brand's personality must be audible in every sentence it writes, from a homepage headline to an error message.

Without explicit tone guidelines, brand voice drifts. The founder writes the landing page in one style, a contractor writes onboarding emails in another, and a support agent responds in a third. Customers experience these inconsistencies as a lack of identity — the brand feels like a committee, not a character. Tone dimensions solve this by giving every writer a shared calibration tool. Instead of vague instructions like "be friendly," dimensions specify exactly where on each scale the brand lives and what that sounds like in practice through concrete do/don't examples.

This framework matters for startups specifically because early-stage brands write high volumes of copy across many touchpoints before they hire a dedicated copywriter. The founder, the designer, the developer, and eventually an AI agent all produce text. Tone of Voice Guidelines ensure that regardless of who writes, the brand sounds like one person with a consistent personality. The guide feeds directly into M4 prototypation (prototype copy), M5 market validation (consistent test messaging), and M6 MVP (production copy standards).

## Task 1: Define 3-5 Tone Dimensions

**Goal:** Choose tone dimensions relevant to your brand and position each on a slider scale, grounding every choice in Brand Archetypes and Brand Prism personality.

**Inputs:**

| Input | Source | Required |
|-------|--------|----------|
| Brand Archetypes analysis (primary and secondary archetypes) | `brand_archetypes.md` applied framework | Yes |
| Brand Prism (Personality facet) | `brand_prism.md` applied framework | Yes |
| Golden Circle (Why statement) | `golden_circle.md` applied framework | Yes |
| Messaging Architecture (brand promise, key messages) | `messaging_architecture.md` applied framework | Yes |
| JTBD analysis (customer language patterns, emotional context) | `jtbd.md` applied framework | No |

**Action:**

1. Review the Brand Archetypes primary archetype. Each archetype implies natural tone tendencies:
   - Hero/Explorer: confident, direct, action-oriented.
   - Sage/Creator: thoughtful, precise, inventive.
   - Caregiver/Everyman: warm, accessible, supportive.
   - Rebel/Magician: bold, surprising, transformative.
   - Use these as starting intuitions, not rigid rules.
2. Review the Brand Prism Personality facet. List the 3-5 personality traits defined there (for example, "curious, precise, encouraging, direct"). These traits must be audible in the tone.
3. Select 3-5 tone dimensions from this menu (or define your own if none fit):
   - **Formal ---- Casual:** How structured and conventional is the language?
   - **Serious ---- Playful:** How much levity and humor does the brand use?
   - **Humble ---- Confident:** How assertive is the brand about its claims?
   - **Technical ---- Accessible:** How much domain expertise does the language assume?
   - **Reserved ---- Enthusiastic:** How much energy and emotion does the brand express?
   - **Respectful ---- Irreverent:** How much does the brand challenge conventions?
   - **Abstract ---- Concrete:** How specific and tangible is the language?
4. For each chosen dimension, position the brand on the scale. Use a 1-5 scale where 1 is the left pole, 3 is neutral, and 5 is the right pole. Do not default to 3 on every dimension — a brand that is neutral on everything has no personality.
5. For each position, write a one-sentence rationale connecting it to the Brand Archetype and/or Prism Personality. For example: "Positioned at 4 on Confident because the Hero archetype speaks with conviction, and our Prism personality includes 'assertive.'"
6. Cross-check all dimensions together. Read them as a set and ask: does this combination describe a coherent person? If the dimensions contradict each other (for example, highly formal AND highly playful), resolve the tension or explain when each takes precedence.

**Output:**

| Output | Format | Purpose |
|--------|--------|---------|
| Tone dimension sliders | Table: Dimension, Scale Position (1-5), Rationale | Defines the measurable verbal personality of the brand |
| Dimension summary narrative | 2-3 sentences describing the overall voice | Quick-reference for anyone writing brand copy |

**Validation:**

- [ ] 3-5 tone dimensions are defined, each with an explicit scale position (not all neutral/3).
- [ ] Each dimension position has a rationale connecting to Brand Archetypes or Brand Prism Personality.
- [ ] The dimensions as a set describe a coherent, distinctive personality (not generic or contradictory).
- [ ] The dimension set can distinguish this brand from at least two competitors (a competitor would position differently on at least 2 dimensions).
- [ ] The Golden Circle Why is audible in the overall tone — the emotional register matches the brand's purpose.

## Task 2: Write Do/Don't Examples Per Dimension

**Goal:** For each tone dimension, produce 3 pairs of do/don't examples using real brand scenarios to show what the tone sounds like in practice.

**Inputs:**

| Input | Source | Required |
|-------|--------|----------|
| Tone dimension sliders | Task 1 | Yes |
| Messaging Architecture (key messages, proof points) | `messaging_architecture.md` applied framework | Yes |
| Brand scenarios (welcome email, error message, feature announcement, support response) | Standard touchpoints | Yes |
| JTBD analysis (customer language) | `jtbd.md` applied framework | No |

**Action:**

1. Define four brand scenarios that every startup encounters. These will recur across examples:
   - **Welcome email:** First communication after signup. Sets the relationship tone.
   - **Error message:** Something went wrong. Tests empathy and clarity under pressure.
   - **Feature announcement:** New capability shipped. Tests enthusiasm and clarity.
   - **Support response:** Customer asked for help. Tests helpfulness and personality.
2. For each tone dimension, write 3 do/don't pairs. Each pair shows the same content expressed two ways:
   - **Do:** Copy that matches the dimension position (the correct tone).
   - **Don't:** Copy that violates the dimension position (the wrong tone for this brand).
3. Distribute examples across the four scenarios so that each scenario appears at least twice across all dimensions. This ensures the guide covers real situations, not abstract concepts.
4. For each pair, add a one-line explanation of why the "Do" version is correct. Reference the dimension and its position. For example: "The 'Do' version is direct and action-oriented (Confident: 4/5), while the 'Don't' version hedges and undermines conviction."
5. Use actual brand terminology and messaging from the Messaging Architecture where possible. The examples should feel like real copy the brand would produce, not generic placeholder text.
6. Read all examples together as a set. Ask: would a new writer, reading only these examples, be able to write on-brand copy for a new scenario? If not, add more examples or sharpen the explanations.

**Output:**

| Output | Format | Purpose |
|--------|--------|---------|
| Do/don't example pairs | 3 pairs per dimension with scenario labels and explanations | Shows what the tone sounds like in practice; the primary training tool for writers |
| Scenario coverage matrix | Table showing which scenarios appear in which dimensions | Ensures examples cover real brand situations comprehensively |

**Validation:**

- [ ] Each dimension has exactly 3 do/don't pairs (not fewer).
- [ ] Each pair uses a real brand scenario (welcome email, error message, feature announcement, or support response).
- [ ] Each pair has a one-line explanation referencing the dimension and its position.
- [ ] All four scenarios appear at least twice across the full set of examples.
- [ ] A writer unfamiliar with the brand could read the examples and produce on-brand copy for a new scenario.

## Task 3: Create Context-Specific Tone Adjustments

**Goal:** Define how tone shifts by communication context while the core personality stays constant. The personality does not change; the intensity adjusts.

**Inputs:**

| Input | Source | Required |
|-------|--------|----------|
| Tone dimension sliders | Task 1 | Yes |
| Do/don't examples | Task 2 | Yes |
| Messaging Architecture (audience cards, journey stages) | `messaging_architecture.md` applied framework | Yes |
| Brand Prism (Relationship facet) | `brand_prism.md` applied framework | No |

**Action:**

1. Define five communication contexts and the emotional state of the reader in each:
   - **Marketing (website, ads, social):** Reader is browsing, deciding whether to pay attention. Emotional state: distracted, skeptical. Tone adjustment: dial up aspiration and personality to earn attention.
   - **Onboarding (welcome flows, setup guides):** Reader just committed and is learning. Emotional state: hopeful but uncertain. Tone adjustment: dial up encouragement and clarity, dial down complexity.
   - **Error/Support (error messages, support responses):** Reader is frustrated or confused. Emotional state: anxious, possibly annoyed. Tone adjustment: dial up empathy and directness, dial down playfulness and enthusiasm.
   - **Documentation (help docs, technical guides):** Reader needs specific information. Emotional state: task-focused, impatient. Tone adjustment: dial up precision and accessibility, dial down personality and flair.
   - **Social media (posts, replies, community):** Reader is in a social, informal environment. Emotional state: casual, open to personality. Tone adjustment: dial up personality and enthusiasm, allow more informality than other contexts.
2. For each context, create a **tone adjustment table** that shows how each dimension shifts relative to the baseline:
   - Use notation like "+1" (slightly more toward right pole), "-1" (slightly more toward left pole), or "0" (no change).
   - No adjustment should exceed +/- 1 from baseline. The personality does not transform; it modulates.
3. Write a one-sentence summary for each context describing the overall tone shift. For example: "In error/support contexts, we stay confident but become more empathetic and direct — we acknowledge the frustration, state what happened, and tell the user exactly what to do next."
4. Identify the **non-negotiable core** — the 1-2 dimensions that never shift regardless of context. These are the brand's most defining traits. For example: "We are always accessible (Technical-Accessible: 4/5) and always confident (Humble-Confident: 4/5), regardless of context."
5. Flag any context where the tone adjustment might create tension with the Brand Prism Relationship facet. For example, if the Prism defines the relationship as "trusted advisor," ensure that even social media contexts maintain that trust dynamic.

**Output:**

| Output | Format | Purpose |
|--------|--------|---------|
| Context-tone adjustment matrix | Table: Context, Dimension Adjustments (+/- 1), Summary | Shows how each dimension shifts per context while preserving core personality |
| Non-negotiable core | 1-2 dimensions that never shift | Defines the absolute minimum brand personality that appears everywhere |
| Context summaries | One sentence per context | Quick-reference for writers working in a specific context |

**Validation:**

- [ ] All five contexts (marketing, onboarding, error/support, documentation, social media) have defined tone adjustments.
- [ ] No adjustment exceeds +/- 1 from the baseline dimension position.
- [ ] 1-2 non-negotiable core dimensions are identified and documented.
- [ ] Context summaries describe the overall tone shift in practical, specific terms (not vague instructions like "be nice").
- [ ] Error/support context explicitly addresses empathy and directness — this is where brands most commonly fail.

## Task 4: Write Sample Copy For 5 Common Scenarios

**Goal:** Produce reference copy (3-5 sentences each) for five common brand scenarios that demonstrate the full tone system in action.

**Inputs:**

| Input | Source | Required |
|-------|--------|----------|
| Tone dimension sliders | Task 1 | Yes |
| Do/don't examples | Task 2 | Yes |
| Context-tone adjustments | Task 3 | Yes |
| Messaging Architecture (brand promise, key messages, CTAs) | `messaging_architecture.md` applied framework | Yes |
| Brand Archetypes (primary archetype narrative style) | `brand_archetypes.md` applied framework | No |

**Action:**

1. Write sample copy for these five scenarios. Each piece must be 3-5 sentences:
   - **Homepage headline + subhead:** The first words a visitor reads. Must express the brand promise in the marketing tone. Pull the headline from Messaging Architecture's brand promise; write a subhead that adds specificity.
   - **Welcome email:** Sent immediately after signup. Must express encouragement in the onboarding tone. Reference what the user just did and what happens next.
   - **Error message:** Displayed when something fails (payment error, data import failure, connection timeout). Must express empathy and clarity in the error/support tone. State what happened, what it means, and what to do.
   - **Feature update announcement:** Sent or posted when a new capability ships. Must express enthusiasm in the marketing tone. Name the capability, state the benefit, and include a CTA.
   - **Customer success story intro:** The opening paragraph of a case study or testimonial page. Must express credibility in the marketing tone. Introduce the customer, their challenge, and the outcome.
2. For each piece, annotate which tone dimensions and context adjustments you applied. This makes the sample copy a teaching tool, not just an example.
3. For each piece, note which Messaging Architecture elements you drew from (brand promise, specific key message, specific proof point, specific CTA).
4. Read all five pieces sequentially. Ask: do they sound like the same brand speaking in different situations? If any piece sounds like a different company wrote it, revise until the personality is consistent.
5. Have the brand promise audible (directly or indirectly) in at least 3 of the 5 pieces. The promise does not need to appear verbatim, but the core idea must be present.

**Output:**

| Output | Format | Purpose |
|--------|--------|---------|
| 5 sample copy pieces | 3-5 sentences each with annotations | Reference copy for M4 prototype, M5 test messaging, and M6 production standards |
| Annotation notes | Per-piece notes on dimensions, context, and messaging sources | Teaching tool showing how the tone system produces real copy |

**Validation:**

- [ ] All five scenarios have sample copy (3-5 sentences each).
- [ ] Each piece has annotations showing which tone dimensions, context adjustments, and Messaging Architecture elements were applied.
- [ ] All five pieces sound like the same brand speaking in different situations (consistent personality, varied intensity).
- [ ] The brand promise is audible in at least 3 of the 5 pieces.
- [ ] The error message demonstrates empathy and clarity without losing brand personality.
- [ ] A new writer could use these samples as templates for similar scenarios.

## Task 5: Validate Consistency And Wire Into M4/M5

**Goal:** Test all outputs for consistency with upstream brand frameworks and connect the Tone of Voice guide to downstream execution frameworks.

**Inputs:**

| Input | Source | Required |
|-------|--------|----------|
| All Tasks 1-4 outputs (dimensions, examples, adjustments, sample copy) | Tasks 1-4 | Yes |
| Brand Prism (all facets) | `brand_prism.md` applied framework | Yes |
| Brand Archetypes (primary archetype) | `brand_archetypes.md` applied framework | Yes |
| Brand Positioning Statement | `brand_positioning.md` applied framework | Yes |
| Messaging Architecture | `messaging_architecture.md` applied framework | Yes |

**Action:**

1. Run a **Brand Prism consistency check**:
   - Read each Brand Prism facet (Physique, Personality, Culture, Relationship, Reflection, Self-image).
   - For each facet, verify that the tone dimensions and sample copy are compatible. Specifically:
     - Personality facet traits must be audible in the tone dimensions.
     - Relationship facet must be preserved in the context adjustments (especially error/support).
     - Culture facet values must not be contradicted by any sample copy.
   - Document any inconsistencies and resolve them by adjusting the tone guide (the Prism takes precedence).
2. Run an **Archetypes consistency check**:
   - Read the primary archetype description and its typical voice characteristics.
   - Verify that the tone dimensions align with the archetype's natural voice. A Sage brand should not sound like a Rebel; a Caregiver brand should not sound like a Hero.
   - Document alignment or adjustments made.
3. Run a **Positioning consistency check**:
   - Read the positioning statement.
   - Verify that the tone would feel natural delivering the positioning claim. If the positioning is about precision and reliability, the tone should not be recklessly playful.
4. Assemble the complete **Tone of Voice Guide** as a single document containing:
   - Tone dimension sliders with rationale.
   - Do/don't examples per dimension.
   - Context-specific adjustment matrix.
   - Non-negotiable core dimensions.
   - Sample copy for 5 scenarios with annotations.
   - Consistency check results.
5. Document explicit wiring to downstream frameworks:
   - **M4 Prototypation:** All prototype copy (landing page, onboarding flow, UI microcopy) must follow this guide. Reference the sample copy as starting templates.
   - **M4 Design Brief:** The tone dimensions inform the verbal identity direction in the design brief. Visual and verbal identity must align.
   - **M5 Market Validation:** All test messaging (A/B tests, SPIN scripts, smoke test copy) must be written in this tone. Inconsistent tone in tests pollutes validation results.
   - **M6 MVP:** This guide becomes the production copy standard. All marketing, onboarding, support, and documentation copy follows the dimensions, context adjustments, and examples defined here.
6. Save the document as the applied framework at `projects/[project]/founder/brand/tone_of_voice.md`.
7. Update the M3 founder diary with key tone decisions: which dimensions were chosen, which trade-offs were made, and how the tone connects to the brand personality.
8. Update the project memo Progress > Brand section to note that Tone of Voice is complete and that all M3 Brand frameworks are finished.

**Output:**

| Output | Format | Purpose |
|--------|--------|---------|
| Tone of Voice Guide | Applied framework in `projects/[project]/founder/brand/` | Canonical reference for how the brand sounds across all contexts |
| Consistency check results | Section in the guide | Proves alignment with Prism, Archetypes, and Positioning |
| Downstream wiring notes | Section in the guide | Ensures M4, M5, M6 teams know to follow this guide for all copy |
| Updated founder diary and project memo | Existing project artifacts | Captures tone decisions and marks M3 Brand as complete |

**Validation:**

- [ ] The Tone of Voice Guide contains all components: dimensions, examples, adjustments, sample copy, consistency checks.
- [ ] Brand Prism consistency check passed — all facets are compatible with the tone.
- [ ] Brand Archetypes consistency check passed — the tone aligns with the primary archetype's natural voice.
- [ ] Positioning consistency check passed — the tone feels natural delivering the positioning claim.
- [ ] The guide explicitly references M4, M5, and M6 as downstream consumers.

## Pitfalls

**Defining Tone Without Brand Personality First**

Tone of Voice is the audible expression of the brand's personality, not a standalone exercise. Defining tone dimensions without completing Brand Archetypes and Brand Prism produces arbitrary style preferences with no strategic foundation. Instead: always ground every dimension choice in a specific archetype trait or Prism personality facet. If you cannot trace a dimension to an upstream framework, question whether it belongs.

**Using Vague Tone Descriptions**

Instructions like "be friendly," "be professional," or "be authentic" are meaningless because every brand claims them. Vague descriptions give writers no calibration tool and produce inconsistent copy. Instead: use slider scales with specific positions and do/don't examples. A writer should be able to compare two draft sentences and determine which one matches the dimension position.

**Ignoring Context Adjustments**

A brand that sounds the same in a marketing email and an error message has no emotional intelligence. Customers in different emotional states need different intensity, not different personality. Brands that use the same playful tone when a payment fails as when announcing a new feature damage trust. Instead: define explicit adjustments per context, especially for error/support scenarios where empathy and clarity must increase.

**Writing Examples That Are Too Generic**

Do/don't examples that use placeholder content ("Hello, valued customer! We're excited to share...") teach nothing because they are not specific to the brand. Generic examples produce generic copy. Instead: write examples using the brand's actual product, terminology, and messaging from the Messaging Architecture. The examples should be reusable as draft copy, not just illustrative concepts.

**Treating The Guide As A Constraint Rather Than A Tool**

Some teams create tone guidelines and then find that nobody uses them because they feel restrictive. The problem is framing: tone guidelines exist to make writing faster and better, not to police creativity. Instead: position the guide as a decision-making tool. When a writer is stuck, the dimensions and examples tell them exactly how to sound. Frame the guide as "here is how we sound" not "here is what you must not do."

**Never Updating For New Contexts**

Startups add new communication channels constantly (chatbot, push notifications, video scripts, investor updates). A tone guide written only for email and website becomes irrelevant as the product scales. Instead: build the guide with an extensible structure. New contexts can be added by defining the reader's emotional state and applying the adjustment framework from Task 3.

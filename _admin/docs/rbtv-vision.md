## Elevator Pitch

**The opportunity:** We're entering the Workflows Era (2025-2026) — the third wave of AI integration into society. For the first time, consumer grade AI tools can handle 15-minute context windows, integrate with workflow automation through MCPs, and enable "vibe coding" where ideas translate to working software in hours instead of months.

**The gap:** AI IDEs like Cursor and Lovable execute brilliantly, but they don't know *what* to build. Development frameworks like BMAD (32.6k GitHub stars) solve the technical structure problem with 21 specialized agents. But the business side — strategy, innovation, problem definition — remains unstructured.

**The solution:** Robotville is the business-side analog to BMAD. It provides the structured thinking layer that happens *before* development: transforming vague ideas into validated concepts through guided questioning, structured planning with quality gates, and milestone-based project evolution.

**The thesis:** The structural drop in building and operating digital products without being technical — enabled by no-code tools and AI — makes 1-5 person businesses viable at $50k/month in dividends. Brazil's infrastructure (Pix, Pix Automático, WhatsApp, local e-commerce platforms) provides an ideal testing ground, but the thesis is globally applicable.

**The founder:** Business background (IB, VC, corporate development) — explaining why Robotville focuses on innovation structure rather than development execution.

## The Opportunity — Vibe Coding and the Innovation Gap

### Defining Vibe Coding

Vibe coding is not "talk to AI and get code." It's **planning-heavy, execution-light development** where humans stay in the loop for high-impact decisions.

The pattern:
1. Humans do the strategic thinking and planning
2. AI executes the implementation
3. Humans review checkpoints and validate outputs
4. Repeat with increasing confidence

This approach produces dramatically better results than prompt-and-pray. When AI covers the full cycle (requirements → code → test → publish) rather than just the coding step, gains become exponential rather than incremental.

### Robotville Core Concepts

Robotville's system is built on five foundational principles:

#### 1. Context Engineering
**"Context is everything with AI"** — Provide structured, relevant context that guides AI focus and decision-making.[^1]

#### 2. Planning-Heavy Development
**"Plan first, execute second"** — Invest time in structured planning before implementation to ensure coherent, purposeful outputs.[^2]

#### 3. Human-in-the-Loop for Critical Decisions
**"Humans decide what matters most"** — Keep humans involved in high-impact decisions (strategy, validation, quality gates) while delegating execution.[^3]

#### 4. Automated Execution with Iteration (Compound Engineering)
**"AI executes and improves the system"** — Automate not just implementation but also system refinement through compound engineering principles, allowing the system to evolve based on learnings.[^2]

#### 5. Tool-Enabled AI Agency
**"Give AI the tools humans would use"** — Equip AI agents with comprehensive tooling (development, testing, deployment, system management) and teach them human-equivalent execution patterns.[^2]

### The Problem: Execution Without Direction

AI tools like Cursor and Lovable are powerful executors. But execution without direction produces technically correct outputs that don't connect to a coherent whole.

The gap: **AI knows how to build, but doesn't know what to build.**

Without structure, chaos emerges:
- Inconsistent patterns
- Forgotten conventions
- Documentation drift
- Agent confusion
- Projects that work technically but don't solve real problems

### BMAD Solves Development Structure

[BMAD](https://github.com/bmad-code-org/BMAD-METHOD) (Breakthrough Method of Agile AI-Driven Development) addresses the development structure gap:

| Aspect | Detail |
|--------|--------|
| Traction | 32.6k GitHub stars, 4.2k forks |
| Architecture | 21 specialized agents, 50+ guided workflows |
| Philosophy | "Traditional AI tools do the thinking for you, producing average results. BMad agents act as expert collaborators who guide you through a structured process to bring out your best thinking." |
| Planning path | product-brief → PRD → architecture → epics/stories → sprint planning → dev |

BMAD provides the structure that makes AI development effective. But it focuses on the *development* side — the technical execution of a defined solution.

### The Missing Layer: Business and Innovation Structure

BMAD assumes you already know:
- What problem you're solving
- Who you're solving it for
- Why your solution matters
- How the business model works

For technical founders, this assumption holds. For business founders — or for anyone starting from a vague idea rather than a clear spec — the business and innovation structure is the bottleneck.

## Robotville — The Business-Side Structure Layer

### What Robotville Is

Robotville is the **business-focused analog to BMAD**. Where BMAD provides structure for the development phase, Robotville provides structure for the innovation and strategy phase that happens *before* development.

| Layer | Focus | Tool |
|-------|-------|------|
| Business/Innovation | What to build, for whom, why it matters | **Robotville** |
| Development | How to build it technically | BMAD |
| Execution | Actually writing the code | Cursor, Claude Code, Lovable |

Robotville doesn't make decisions for you. It **asks the right questions so you make better decisions**.

### Timeline: Independent Development

The approach emerged independently before exposure to existing frameworks:

| Date | Milestone |
|------|-----------|
| September 2025 | Started macro thesis (one-person business opportunity) |
| November 2025 | Started Robotville specifically |
| January 2026 | Discovered BMAD |

This timeline matters: Robotville's philosophy of structured thinking with human-in-the-loop emerged from first principles, not from copying existing approaches. The convergence with BMAD's philosophy validates the underlying insight.

### Core Philosophy: Human-in-the-Loop

Robotville's approach to AI collaboration:

| Stage | Human Role | AI Role |
|-------|------------|---------|
| Direction | Set goals and constraints | Clarify options |
| Planning | Approve and refine | Propose and draft |
| Execution | Review checkpoints | Execute tasks |
| Validation | Test end results | Report outcomes |

The system is designed around planning-heavy, execution-light principles where humans participate in decisions with the biggest impact.

### Core Components

| Component | Function |
|-----------|----------|
| **domcobb** | Transforms vague problems into structured solutions through iterative questioning (Socratic method) |
| **Planning Workflow** | Structured plan creation and execution with quality gates (phases, checkpoints, condensation) |
| **Judge Sub-Agent** | Independent quality evaluator that ensures work meets standards before completion |
| **Founder Module** | Six milestones from napkin sketch to validated MVP, guided by the **mentor** agent |
| **Map System** | Use-case-based navigation files that route agents to relevant resources while minimizing context load |

### The Founder Module

The founder module guides projects from raw idea to launch-ready MVP through 6 milestones:

| # | Milestone | Goal |
|---|-----------|------|
| M1 | Conception | Structure idea into comprehensive business concept |
| M2 | Validation | Validate technical and financial feasibility |
| M3 | Brand | Create preliminary brand book |
| M4 | Prototypation | Build working HTML prototype for F&F testing |
| M5 | Market Validation | Validate market demand and pricing |
| M6 | MVP | Create MVP usable by real clients |

Each milestone uses frameworks from business strategy (Jobs to Be Done, Working Backwards) adapted for AI-assisted execution.

### Founder Background

Robotville was built by a founder with:
- Investment banking experience
- Venture capital experience
- Corporate development experience
- Non-technical background (business-focused)

This explains the focus: a technical founder would optimize for development efficiency. A business founder optimizes for innovation structure — the thinking that happens before code exists.

### Platform Vision

Robotville currently operates on Cursor, but the approach is platform-agnostic:

| Platform | Use Case | Status |
|----------|----------|--------|
| Cursor | Primary IDE for development | Current |
| Claude Code | Anthropic's agentic coding tool | Future |
| Lovable | Frontend development via GitHub integration | Future |
| Agentic files | Mobile apps (ChatGPT, Gemini, Claude, Manus AI) | Future |

The core insight: **using any AI-powered IDE without context and structure leads to poor results**. Robotville provides that context and structure as portable files that work across platforms.

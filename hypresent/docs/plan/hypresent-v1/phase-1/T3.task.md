# Task T3: Iframe Load + Runtime Boot Stub

## Goal

Load a user document into a same-origin iframe via `/doc/`, and inject a runtime boot stub that exposes `window.hyp` and emits a `ready` event after the iframe loads.

### Phase: Understand
1. MUST read every Context File.
2. Mark T3 in progress in BOTH locations.
3. Confirm: served `/doc/` bytes MUST be identical to disk — runtime is injected AFTER `load`, never server-side. The injected `<script type="module">` uses the ABSOLUTE src `/runtime/js/runtime-main.js` (the fixed `/runtime/*` route from T1), NEVER a `/doc/`-relative path, so its `import` chain resolves against the runtime's own origin.

### Phase: Execute
1. CREATE `app/js/api-client.js`: `open(path)→{html,dir,name}` fetch wrapper (throws on non-2xx with server `{error}`).
2. CREATE `app/js/shell/file-controls.js` (open path only): on open, call `api-client.open`, set `iframe.src=/doc/<name>`.
3. CREATE `runtime/js/runtime-main.js` (boot stub): on iframe `DOMContentLoaded`/`load`, expose `window.hyp` command object (stub), emit `ready` (placeholder payload `{tokens:[],sections:[]}` until T5/T16 fill it).
4. Parent appends to the iframe document, on iframe `load`, a `<script type="module">` with `src="/runtime/js/runtime-main.js"` (ABSOLUTE — origin = server origin; NOT `/doc/`-relative). The runtime's `import`s of sibling modules resolve against `/runtime/js/`; vendored-lib imports use absolute `/app/js/vendor/...`. Post-load injection only.
